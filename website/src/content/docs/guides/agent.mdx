The [Agent concept](/concepts/agent) page covers what an Agent *is*.
This guide is a catalog of the **patterns** you'll reach for when
building one.

## Skeleton

The minimum agent: a class, a name, a system prompt:

```typescript
import * as Alchemy from "alchemy";
const { Agent } = Alchemy.Agent;

export class AssistantAgent extends Agent<AssistantAgent>()("AssistantAgent")`
  You are a concise assistant. Answer in one sentence.
` {}
```

`yield* AssistantAgent` returns an `AgentInstance` with `generateText`
/ `streamText`. The `LanguageModel` is *not* baked in — it's provided
at each call site.

## Calling the agent

```typescript
const assistant = yield* AssistantAgent;
const response = yield* assistant
  .generateText({ prompt: "Summarize Effect's `Stream` in one line." })
  .pipe(Effect.provide(OpenAiModel));
```

Switching providers is one line at the call site — every `Agent` in
your app keeps working without edits:

```diff lang="typescript"
- .pipe(Effect.provide(OpenAiModel));
+ .pipe(Effect.provide(AnthropicModel));
```

## Adding tools

A `Tool` is a class whose tagged-template prompt fragment includes
the parameter / success schemas inline. **Inline form** — class +
prompt + handler in one declaration:

```typescript
import { Tool } from "alchemy/Agent";
import * as S from "effect/Schema";

export class CreateObject extends Tool("CreateObject")`
  Create an object in the bucket.
  Return ${{ key: S.String }}
`(
  Effect.gen(function* () {
    const bucket = yield* Cloudflare.R2Bucket.bind(Bucket);
    return Effect.fnUntraced(function* (message) {
      return yield* bucket.put(`${message}.txt`, "data");
    });
  }),
) {}
```

Reference the tool inside the agent's prompt with `${CreateObject}`:

```typescript
export class Blogger extends Agent<Blogger>()("Blogger")`
  When asked to, use the ${CreateObject} tool to echo a message back.
` {}
```

The agent's instance is now configured with that tool — the model can
call it during `generateText` / `streamText`.

## Separating tool tags from layers

For larger codebases, declare the tool's *type* in one file and its
*implementation* in another. Use the **tag-only form**:

```typescript
// tools/create-object.ts — types only, importable from anywhere
export class CreateObject extends Tool<CreateObject>()("CreateObject")`
  Create an object in the bucket.
  Return ${{ key: S.String }}
` {}
```

```typescript
// tools/create-object.live.ts — runtime layer
import { CreateObject } from "./create-object.ts";

export default CreateObject.make(
  Effect.gen(function* () {
    const bucket = yield* Cloudflare.R2Bucket.bind(Bucket);
    return Effect.fnUntraced(function* (message) {
      return yield* bucket.put(`${message}.txt`, "data");
    });
  }),
);
```

The agent's prompt still references `${CreateObject}` (the tag); the
runtime `.make(...)` layer is provided wherever the agent runs.

## Hosting in a Durable Object

For per-conversation memory, host the agent inside a Durable Object.
The DO instance *is* the conversation; `getByName(threadId)` selects
a thread:

```typescript
export class BloggerObject extends Cloudflare.DurableObjectNamespace<BloggerObject>()(
  "BloggerObject",
  Effect.gen(function* () {
    const blogger = yield* Blogger;
    return Effect.gen(function* () {
      return {
        fetch: Effect.gen(function* () {
          const response = blogger.streamText({ prompt: "Hello World" }).pipe(
            Stream.flatMap((part) =>
              part.type === "text-delta"
                ? Stream.fromArray([new TextEncoder().encode(part.delta)])
                : Stream.empty,
            ),
            Stream.provide(OpenAiModel),
          );
          return HttpServerResponse.stream(response, { status: 200 });
        }),
      };
    });
  }),
) {}
```

Layer
[`Cloudflare.DurableObjectChatPersistence`](/tutorial/cloudflare/durable-object-chat-persistence)
underneath Effect AI's `Persistence.layerResultPersisted({ storeId })`
to persist chat history across restarts and hibernation.

## Picking a model

`OpenAiModel` (or `AnthropicModel`, or `aiGateway.model({ ... })`) is
just a `LanguageModel` `Layer`. Build it once and provide it at every
call site:

```typescript
import { OpenAiClient, OpenAiLanguageModel } from "@effect/ai-openai";
import { Config, Layer } from "effect";
import { FetchHttpClient } from "effect/unstable/http";

const OpenAiClientLayer = OpenAiClient.layerConfig({
  apiKey: Config.redacted("OPENAI_API_KEY"),
}).pipe(Layer.provide(FetchHttpClient.layer));

export const OpenAiModel = OpenAiLanguageModel.model("gpt-5.2").pipe(
  Layer.provide(OpenAiClientLayer),
);
```

For Workers AI through a Cloudflare AI Gateway, use
`aiGateway.model({ ... })` (see the
[AI Gateway tutorial](/tutorial/cloudflare/ai-gateway)). The
`Effect.provide(...)` call site doesn't change.

## Streaming responses

`streamText` returns a `Stream` of typed parts. The common pattern is
to forward `text-delta` parts as SSE bytes:

```typescript
fetch: Effect.gen(function* () {
  const blogger = yield* Blogger;
  const response = blogger.streamText({ prompt }).pipe(
    Stream.flatMap((part) =>
      part.type === "text-delta"
        ? Stream.fromArray([new TextEncoder().encode(part.delta)])
        : Stream.empty,
    ),
    Stream.provide(OpenAiModel),
  );
  return HttpServerResponse.stream(response, { status: 200 });
}),
```
