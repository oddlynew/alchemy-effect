An **Agent** is a class whose system prompt is a tagged template
literal. Tools are referenced inline in the prompt with `${ToolClass}`,
and the class identity is the tag you `yield*` to call the agent:

```typescript
export class Blogger extends Agent<Blogger>()("Blogger")`
  When asked to, use the ${CreateObject} tool to echo a message back.
` {}

// later, anywhere in a Worker / DO
const blogger = yield* Blogger;
blogger.streamText({ prompt: "Hello World" });
```

The `Agent` class bundles two things: the **system prompt** and the
set of **tools** the model can call. The `LanguageModel` is *not*
baked into the agent — it's provided to each `generateText` /
`streamText` call. That's deliberate: every call site can pick its
own model (Workers AI through an AI Gateway, OpenAI, Anthropic, …)
without rewriting the agent.

## Tools

A `Tool` is a class whose template literal is the prompt fragment the
model sees, with parameter / return schemas interpolated inline:

```typescript
export class CreateObject extends Tool("CreateObject")`
  Create an object in the bucket.
  Return ${{ key: S.String }}
`(
  Effect.gen(function* () {
    const bucket = yield* Cloudflare.R2Bucket.bind(Bucket);
    return Effect.fnUntraced(function* (message) {
      return yield* bucket.put(`${message}.txt`, "data");
    });
  }),
) {}
```

`${{ key: S.String }}` declares the success schema. Parameter schemas
go in the same place. The handler is an `Effect.fnUntraced` that
receives the model's call arguments.

There's also a **tag-only form** for separating declaration from
implementation:

```typescript
// types and schema only — safe to import from anywhere
export class CreateObjectTag extends Tool<CreateObject>()("CreateObject")`
  Create an object in the bucket.
  Return ${{ key: S.String }}
` {}

// the runtime implementation — provided as a Layer
export default CreateObjectTag.make(
  Effect.gen(function* () {
    const bucket = yield* Cloudflare.R2Bucket.bind(Bucket);
    return Effect.fnUntraced(function* (message) {
      return yield* bucket.put(`${message}.txt`, "data");
    });
  }),
);
```

The agent's prompt references the tag class (`${CreateObject}`); the
runtime layer is provided wherever the agent is hosted.

## Where Agent fits

Effect's `unstable/ai` module gives you:

- **`LanguageModel`** — the provider-agnostic `generateText` /
  `streamText` API.
- **`Toolkit`** — typed tool definitions and their handlers.
- **`Prompt`** — message construction.
- **`Chat`** — turn-by-turn conversation state with persistence.

`Alchemy.Agent` sits on top of those: the class is the prompt + the
toolkit, the `LanguageModel` layer is provided at the call site, and
chat persistence is wired up by the surrounding host (typically a
`DurableObjectNamespace`).

## Composition with other concepts

| Concept | How Agent uses it |
| --- | --- |
| [Resource](/concepts/resource) | The agent's host (a `Worker`, a `DurableObjectNamespace`, an `AiGateway`) is a resource. The agent itself is just a class. |
| [Binding](/concepts/binding) | Tools usually bind to resources (`Cloudflare.R2Bucket.bind(Bucket)`, `Cloudflare.D1Database.bind(DB)`) inside their handler Effect. |
| [Layer](/concepts/layers) | `LanguageModel` is provided to each call as a Layer; tag-form tools provide their handler as a Layer via `.make`. |
| [Platform](/concepts/platform) | A Worker or DO is the runtime that hosts the agent and provides the `LanguageModel`. |

## DO-backed Agents

Inline agents run inside a Worker's request handler. For
**per-conversation memory**, host the agent in a Durable Object so
each `getByName(threadId)` is its own chat:

```typescript
export class BloggerObject extends Cloudflare.DurableObjectNamespace<BloggerObject>()(
  "BloggerObject",
  Effect.gen(function* () {
    const blogger = yield* Blogger;
    return Effect.gen(function* () {
      return {
        fetch: Effect.gen(function* () {
          const response = blogger.streamText({ prompt: "Hello World" }).pipe(
            Stream.flatMap((part) =>
              part.type === "text-delta"
                ? Stream.fromArray([new TextEncoder().encode(part.delta)])
                : Stream.empty,
            ),
            Stream.provide(OpenAiModel),
          );
          return HttpServerResponse.stream(response, { status: 200 });
        }),
      };
    });
  }),
) {}
```

Add `Cloudflare.DurableObjectChatPersistence` underneath an Effect AI
`Persistence.layerResultPersisted({ storeId })` for chat history that
survives restart and hibernation. See
[Persist conversation state](/tutorial/cloudflare/durable-object-chat-persistence).

## When to reach for an Agent

Use an `Agent` class when:

- You want one canonical place that owns "this LLM's prompt + tools",
  reused across multiple call sites.
- You expect the same agent to be wired to different `LanguageModel`s
  in different contexts (per-environment, per-user-tier, A/B tests).
- The prompt and tools belong together conceptually (the prompt
  references the tools via `${Tool}`).

If you're issuing a one-off `generateText` from a Worker route with no
tools and no reuse, skip `Agent` — just call
`LanguageModel.generateText(...)` directly with a provided model.
